# 简报看板 — 部署方案

## 架构

```
用户浏览器
  │
  ├─ https://voadge.top:668/dashboard/briefing-today.html (Shell HTML)
  │     │
  │     ├─ 读取 localStorage (NOCOBASE_TOKEN / nb_token cookie)
  │     ├─ GET /api/briefings:list?filter=... (今日未读简报)
  │     ├─ 客户端渲染卡片、轮播、进度条
  │     └─ 点击卡片 → GET /api/construction_daily_log:get (弹窗详情)
  │
  └─ NocoBase 后台 (iframe 区块嵌入)
        └─ iframe src = /dashboard/briefing-today.html
```

## 文件清单

| 文件 | 路径 | 说明 |
|------|------|------|
| Shell HTML | `/opt/noco-base/dashboard/briefing-today.html` | 客户端渲染的静态页面，Nginx 直接 serve |
| Shell 模板 | `/opt/noco-base/dashboard/briefing-shell.html` | Worker 启动时若无目标文件则从此拷贝 |
| Worker | `/opt/noco-base/worker/briefing-worker.js` | Node.js 轮询服务，扫描施工日志并创建简报 |
| Nginx 配置 | `/opt/noco-base/nginx.conf` | 含静态文件路由、缓存控制、反向代理 |

## Shell HTML 规范

### 页面结构
- `#app` — 主渲染区（tabs + cards-stage + progress-bar）
- `#modalOverlay > #modalContent` — 弹窗

### 数据获取
```
GET /api/briefings:list?pageSize=50&sort=createdAt&filter={"briefing_date.$dateOn":"2026-06-13"}
Authorization: Bearer <token>
```
- token 从 localStorage 获取：`NOCOBASE_TOKEN` / `nocobase_token` / `token` / `auth_token` / `access_token`
- 兜底 cookie：`nb_token`
- 按 `briefing_type` 分组，只显示**今天日期 + 未读**的简报

### 交互流程
```
自动模式 (默认):
  Tab1: 施工日志 → 卡片列表自动上滚 50px/s → 到底等待 1.5s → Tab2
  Tab2: 安全简报 → 同上 → Tab3 → ... → 循环

点击模式 (用户点击 Tab):
  停在该分类，卡片继续循环滚动
  点击其他 Tab → 切换过去，继续滚动
  无自动恢复（用户行为优先）

卡片:
  鼠标悬停 → 阴影提升
  点击 → 打开 modal（请求 source_workflow_id 对应记录）
  进入视口 15s → 标记已读（localStorage），未读数-1，卡片变淡
```

### 视觉规范

```
CSS 变量 (随系统色调):
  浅色:  bg=#ffffff  card-bg=#ffffff  text=#1f2937  accent=#3b82f6
  深色:  bg=#1a1a2e  card-bg=rgba(255,255,255,0.03)  text=#e0e6ed  accent=#60a5fa

卡片:
  白底 + 圆角 8px + 左侧 3px accent 竖线
  padding: 12px / margin-bottom: 8px
  标题: 1em bold / 时间: 0.75em muted / 摘要: 0.85em #4b5563

弹窗 (NocoBase 风格):
  header: 标题左 + 关闭 X 右
  body: 行式布局 (label 80px + value flex)
  无滚动条，自动撑高
```

### 过期清理
- localStorage key: `briefing_seen_{userId}`
- 存储格式: `{"简报id":"2026-06-13T10:00:00Z", ...}`
- 自动清理超过 7 天的记录

## Worker 轮询服务

### 运行方式
- systemd service: `nocobase-briefing-worker.service`
- 轮询间隔: 30s
- Node.js 脚本，通过 Docker 内部端口 `localhost:80` 调用 API

### 流程
```
启动 → 登录 (POST /api/auth:signIn) → 生成 Shell HTML (仅首次)
        → 轮询:
          GET /api/construction_daily_log:list
          filter: status ∉ [已发布, 已归档, 已提交]
          →
          已复核 → 创建 briefing (title, summary, source_workflow_id=log.id)
                 → 更新 status = 已发布
                 → 通知部门用户
          已审核(无意见) → 通知部门用户
```

### 摘要格式
```
天气:晴 | 施工:E2E测试 | 投入:挖掘机2台,碎石50方,8人 | 安全:无 | 明日:继续作业
```
- fields: `天气` / `施工` / `投入` (设备+材料+人员) / `安全` / `明日`
- 总长度 ≤ 100 字，超长截断

### 关键参数
| 变量 | 值 |
|------|-----|
| POLL_INTERVAL | 30000ms |
| BASE | https://localhost:668 |
| HTML_PATH | /opt/noco-base/dashboard/briefing-today.html |
| SHELL_TEMPLATE | /opt/noco-base/dashboard/briefing-shell.html |

## Nginx 配置

### `/dashboard/briefing-today.html` (精确匹配)
```
expires -1
Cache-Control: no-cache, must-revalidate
Pragma: no-cache
```
- 关键：此 location 必须在 `location /dashboard/` 之前，nginx 长 prefix 优先

### `/dashboard/` (通用静态文件)
```
root /usr/share/nginx/html/dashboard/ (挂载自 host /opt/noco-base/dashboard/)
expires 7d
```

## 部署命令速查

```bash
# SSH
ssh -i E:\voadge.pem ubuntu@110.42.236.231

# 更新 Shell HTML
sudo cp /tmp/briefing-shell.html /opt/noco-base/dashboard/briefing-today.html

# 更新 Worker
sudo cp /tmp/briefing-worker.js /opt/noco-base/worker/briefing-worker.js
sudo systemctl restart nocobase-briefing-worker.service

# 更新 Nginx
sudo cp /tmp/nginx.conf /opt/noco-base/nginx.conf
docker restart noco-base-nginx-proxy-1

# 查看 Worker 日志
journalctl -u nocobase-briefing-worker --no-pager -n 50

# Docker 内部访问 API
docker exec noco-base-app-1 node /tmp/script.js
```
